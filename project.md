# VentureAgent 优化与演进路线图 (Optimization & Roadmap)

本项目旨在打造一个严谨、智能的创业项目全生命周期管理平台。当前已完成实名准入闭环，下一步将聚焦于“多维度联动”与“全局管控”。

---

## 1. 学生端 (Student Workspace)

### 🔍 现有 Bug 检查与加固
- [ ] **附件物理回收**: 
  - **现象**: 当前删除项目后，物理文件仍留在 `uploads/` 目录，虽数据库纪录已删，但硬盘空间未释放。
  - **计划**: 完善 `delete_project_endpoint`，调用 `os.remove` 同步清理，并捕获 `FileNotFoundError` 增强稳健性。
- [ ] **雷达图数据真实化**: 
  - **现象**: 工作台雷达图目前为 Mock 展现。
  - **计划**: 打通 `project_assessments` 表，使学生能直接查看到 AI 审计/导师评分的动态画像。
- [ ] **ID Fallback 清理**: 
  - **现象**: 部分代码中仍存在 `1120230571` 默认值。
  - **计划**: 彻底移除所有 Hardcoded ID，强制走 `localStorage` 获取。

### 🚀 未来功能计划
- **项目类型深度适配**: 针对“互联网+”与“挑战杯”不同的评分标准，动态调整编辑器辅助功能与 DDL 模板。
- **导师在线干预**: 实时接收来自教师端的“指令”弹窗，提供即时修正建议。

---

## 2. 教师端 (Teacher Dashboard)

### 🔍 现有 Bug 检查与加固
- [ ] **大盘数据溢出**: 
  - **现象**: 当全校项目数量极多时，看板渲染性能可能下降。
  - **计划**: 引入分页机制或虚拟列表。
- [ ] **审计结果覆盖**: 
  - **计划**: 确保 DeepSeek 审计在同一项目被多次呼叫时，能正确执行 `ON CONFLICT` 更新，防止由于数据重复导致的图表统计偏差。
- [ ] **实名对齐检查**: 
  - **计划**: 修正 Mock 教师数据的属性（username 应为工号而非姓名），对齐 Neo4j 生产环境变量。

### 🚀 未来功能计划
- **智能风险预警**: 基于 H 原则自动筛选“高危”项目并优先置顶建议。
- **批量导出模块**: 支持一键打包下载所有管辖项目的附件 ZIP。
