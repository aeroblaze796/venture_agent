# 创业智能体前端七阶段优化清单 (task_front.md)

本清单将整个系统的前端开发划分为 7 个核心阶段。每个阶段对应一个独立的功能页面或视图，确保开发过程条理清晰。

---

## 阶段一：【学生端】AI 灵感空间 (AI Workspace)

> 目标：打造最核心的对话互动区，实现导师身份与任务的自动化。

- [X] **1.1 导航框架升级**：在最左侧实现窄条菜单栏，支持在不同模块间自由切换。
- [X] **1.2 导师身份同步**：对接后端接口，根据当前对话自动显示“学习辅助导师”或“项目教练”。
- [ ] **1.3 任务自动提取**：对话中由 AI 布置的任务，自动同步到右侧“待办项”列表。
- [ ] **1.4 规则诊断提醒**：集成规则引擎，对逻辑有误的内容进行“标红”警示并给出建议。

## 阶段二：【学生端】计划书编辑器 (BP Editor)

> 目标：将对话成果转化为结构化的商业计划书（BP）。

- [X] **2.1 结构化写作区**：提供符合竞赛标准的 BP 章节模板（已实现项目 Dashboard 展示层）。
- [ ] **2.2 AI 辅助润色**：根据“AI 灵感空间”的侧边栏建议，一键填充或优化特定章节。
- [X] **2.3 状态关联**：在编辑器中显示项目的实时健康度分值（A+/65% 多维可视化）。

## 阶段三：【学生端】项目协作中心 (Collaboration Hub)

> 目标：展示项目整体进度与团队状态。

- [X] **3.1 进度概览**：可视化展示项目从“选题”到“路演”的里程碑完成度（Timeline 演进轨迹）。
- [ ] **3.2 资源库管理**：存放项目中产生的图表、PPT 和其他参考资料。

## 阶段四：【教师端】班级大盘洞察 (Class Insight)

> 目标：从全局视角监控整个班级的学习进度。

- [X] **4.1 核心指标统计**：展示班级项目数、平均分和进度曲线 (已在 TeacherDashboard 实现)。
- [X] **4.2 高频错误排行**：自动汇总全班学生在对话中触发最多的 Top 5 商业逻辑漏洞 (已集成 H原则 Top Mistakes)。

## 阶段五：【教师端】项目预警中心 (Risk Center)

> 目标：精准锁定风险项目并提供证据。

- [X] **5.1 高危项目清单**：按风险等级自动排列需要人工干预的项目 (Monitoring Inventory 已实装)。
- [X] **5.2 证据链追溯 (Evidence-Trace)**：点击风险点，自动高亮该学生在对话或 BP 中引发报错的原始文本 (集成 Battle Logs 追踪)。

## 阶段六：【教师端】自动化批改台 (Intervention Board)

> 目标：辅助教师完成打分与策略下发。

- [X] **6.1 智能批改建议**：AI 预填 Rubric 评分，教师进行复核或修改 (AI 重新审计功能已闭环)。
- [X] **6.2 干预策略下发**：教师可手动配置干预逻辑，并实时反射到该学生的 AI 对话中 (Teaching Intervention 模块已完成)。

## 阶段七：【管理员】全局管理门户 (Admin Portal)

> 目标：最高权限监控与账户管理。

- [X] **7.1 系统管理员导航入口**：在初始 Portal 页增加“系统管理”入口及权限验证 (AdminMatrix 已集成)。
- [X] **7.2 全校大盘数据**：跨班级、跨学院的项目健康度对比与异常分析。
- [X] **7.3 账号与权限管理**：RBAC 权限分配 (Student/Teacher/Admin 三端 UI 统一)。

## 阶段八：【生态增强】能力闭环与图谱赋能 (GraphRAG & Ecosystem)

> 目标：深化 AI 审计逻辑，构建全栈智能辅助生态。

- [ ] **8.1 GraphRAG 深度集成**：接入 Neo4j 中的项目本体关联，实现基于“价值环路”模型的高阶审计引导。
- [ ] **8.2 多端实时同步加固**：优化 Socket.io 或长轮询机制，确保教师干预指令在学生端 Agent 的即时感知。
- [ ] **8.3 全站移动端响应式适配**：对核心看板（学生/教师/管理）进行流式布局优化，支持移动端管理。
- [ ] **8.4 实名认证审核流**：基于 Neo4j 的身份系统，完善实名认证的审批与驳回业务闭环。
