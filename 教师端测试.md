# VentureAgent (Phase 3) 测试方法与操作指南

经过第三阶段的功能开发，系统的“教师洞察与教学自动化能力”已大幅升级。请遵循以下步骤启动项目并进行端到端测试。

---

## 🚀 1. 项目启动指南

在本地启动前，请确保您在 `venture_agent` 的根目录下，并且已经配置好 [.env](file:///e:/%E5%A4%A7%E4%B8%89%E4%B8%8B/%E5%A4%A7%E6%95%B0%E6%8D%AE%E7%89%B9%E8%89%B2%E8%AF%BE%E7%A8%8B%E6%A8%A1%E5%9D%97/venture_agent/backend/.env) 文件中的大模型环境变量（`DEEPSEEK_API_KEY`）。

### 启动后端服务端 (Terminal 1)
```bash
cd backend
conda activate venture_agent
# 安装缺少的依赖（如不需要可跳过）
pip install -r requirements.txt
# 启动 FastAPI 服务
uvicorn app.main:app --reload --port 8000
```
> 后端通常运行在 `http://127.0.0.1:8000`，您可以访问 `http://127.0.0.1:8000/docs` 查看交互式 API 测试面板（Swagger UI）。

### 启动前端应用端 (Terminal 2)
```bash
cd frontend
# 如果是首次启动需要运行 npm i
npm install
# 启动 Vite 本地测试服务器
npm run dev
```
> 前端通常运行在 `http://localhost:5173`。

## 🧪 2. 核心功能测试链路 (A6 系列)

### 测试案例一：A6-1 单项目精细批改证据链 (Evidence Trace)
1. **操作者身份**：指导老师（测试账密：账号 `T001` 或 `张老师`，密码 `123456`）。
2. **操作动作**：进入某个学生的项目详情页（如果为空，请先用学生账号创建一个含有文字正文的草案）。
3. **验证触发**：点击页面中的“重新评估”或类似的重新审核按钮，此时前端会对给 `/api/teacher/projects/{project_id}/audit` 接口发请求。
4. **预期结果**：系统通过底层大模型读取项目正文后，除了生成1到5分的打分雷达图，还会将提取用来批判学生逻辑的**原文片段（Evidence Trace）**存入后台并跟随展示，实现“判卷有理有据”。

### 测试案例二：A6-2 班级高频错误与干预计划生成
1. **操作者身份**：指导老师 (账号 `T001`)。
2. **操作动作**：点击进入你的“教师控制台 (Teacher Dashboard)”页面。
3. **验证触发 (Top Mistakes)**：控制台加载完后，注意观察“高危项目”或者“高发错误”板块。**这部分已不再是写死的数据**，而是根据您名下所有学生项目的真实AI评审记录动态正则提取并统计出来的次数排名（如 H8单位经济、H1价值错位）。
4. **验证触发 (Teaching Plan)**：在界面或后台找到“生成下周干预方案”逻辑触发点（对应后端 `/api/teacher/intervention-generator`）。系统会自动把全班的所有报错摘要揉合发给教研主任 Agent。
5. **预期结果**：您将得到一份字数控制在 300字 左右的、针对最高频错误的**实操教学计划与建议**。

### 测试案例三：A6-4 对话历史截面与能力画像动态评估
由于目前前端可能暂未绘制该功能的专门面板，极度推荐使用后端的 Swagger 接口测试这个纯 AI 逻辑：
1. **准备数据**：先用一个学生账号（比如测试学号 `1120230571`）进入“灵感空间”或新建会话，并与智能体产生 **3轮及以上** 的实质性对抗问答（比如随意编撰一个无人机送草莓的 BP 让智能体拷问）。
2. **操作者身份**：通过 Swagger UI (`http://127.0.0.1:8000/docs`) 进入。
3. **验证触发**：找到 `GET /api/teacher/projects/{project_id}/capability_profile` 接口。输入刚才那个项目对应的 `project_id` 执行。
4. **预期结果**：系统抽取 SQLite 里 [messages](file:///e:/%E5%A4%A7%E4%B8%89%E4%B8%8B/%E5%A4%A7%E6%95%B0%E6%8D%AE%E7%89%B9%E8%89%B2%E8%AF%BE%E7%A8%8B%E6%A8%A1%E5%9D%97/venture_agent/backend/app/main.py#204-207) 表中倒数6次对话（即 3 轮问答）。大模型将自动按照 `[痛点发现、方案策划、商业建模、资源杠杆、逻辑表达]` 5 个维度输出 JSON 雷达分值，最重要的是：**它一定会在短评中引用该名学生在对话中的原话**（比如：“学生在第一轮提到：『我们的客户是所有人』，缺乏精确细分”）。
